Setting up your vault

Your vault is a private database that only you can access. We use a free service called Supabase to store it securely. You only do this once — after that Hyphælium just works.

🔒

Your data is yours — completely

Supabase is a trusted database service used by hundreds of thousands of developers worldwide. Your vault lives in your own private Supabase project. Hyphælium cannot read your conversations — we only store the address of each one, never the content. You can delete your data at any time.

1
Create a free Supabase account
supabase.com · free forever for personal use

Supabase is free for personal use — no credit card required. You will not be charged anything.

  1. Open a new browser tab and go to supabase.com
  2. Click the green Start your project button
  3. Sign up with your email address — or use your Google account for speed
  4. Check your email and confirm your address if asked
  5. You will land on the Supabase dashboard — a dark screen with your account
Already have a Supabase account? Skip straight to Step 2.
2
Create a new project
This is your private vault container

Each Supabase project is a separate private database. You need one for Hyphælium.

  1. On the Supabase dashboard click New project
  2. Give it a name — Hyphælium is perfect
  3. Create a database password — make it strong, write it down somewhere safe
  4. Choose a region — pick the one closest to you geographically
  5. Click Create new project
  6. Wait about 60 seconds while Supabase sets it up — a progress bar shows
  7. When it says Project is ready — move to Step 3
Important: Save your database password somewhere safe — in a password manager or written down. You will not need it often but you may need it later.
3
Run the Hyphælium setup
One SQL command · takes about 10 seconds

You need to tell your database what to store. This is done by running one block of code — copy and paste, no typing required.

  1. In your Supabase project look at the left sidebar
  2. Click SQL Editor — it looks like a code symbol </>
  3. Click New query
  4. Copy the entire block of code below
  5. Paste it into the large white text area
  6. Click the green Run button at the bottom right
  7. You should see Success. No rows returned — this is correct

Copy this entire block:

CREATE TABLE IF NOT EXISTS hyphaelium_captures ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, captured_at timestamptz NOT NULL DEFAULT now(), platform text NOT NULL, route_path text NOT NULL, identity_tag text NOT NULL, topic text DEFAULT 'general', depth integer DEFAULT 0, boundary_count integer DEFAULT 0, boundary_flag boolean DEFAULT false ); ALTER TABLE hyphaelium_captures ENABLE ROW LEVEL SECURITY; DROP POLICY IF EXISTS "open" ON hyphaelium_captures; CREATE POLICY "open" ON hyphaelium_captures FOR ALL USING (true) WITH CHECK (true); GRANT ALL ON hyphaelium_captures TO anon; GRANT ALL ON hyphaelium_captures TO authenticated; GRANT USAGE ON SCHEMA public TO anon; GRANT USAGE ON SCHEMA public TO authenticated;
Seeing "No rows returned"? That is exactly right. It means the setup worked perfectly. This command creates your vault — it does not return any data.
4
Copy your two keys
You will need these to connect Hyphælium to your vault

Supabase gives you two pieces of information that Hyphælium needs — your Project URL and your API key. These are like an address and a door key for your vault.

  1. In the left sidebar click Settings — the cog icon at the very bottom
  2. Click API in the settings menu
  3. You will see two sections — find Project URL
  4. Click the copy icon next to it — it looks like two overlapping squares
  5. Paste it somewhere temporary — Notepad is perfect
  6. Now find Project API keys
  7. Look for the row labelled anon / public or click Publishable
  8. Click the copy icon next to that key
  9. Paste it below your URL in Notepad

You should now have two things in Notepad that look something like this:

Project URL: https://abcdefghijklmn.supabase.co API Key: sb_publishable_xxxxxxxxxxxxxxxxxxxxxxxx
Which key to use: Use the anon / public key or the Publishable key. Do NOT use the service_role key — that one has full database access and should never leave Supabase.
5
Connect your vault
The last step — then you are done

Now you have your URL and key, you can connect Hyphælium to your vault.

For the bookmarklet: Go to hyphaelium.com/install.html, enter your URL and key, click Connect. Done.

For the vault viewer: Go to hyphaelium.com/vault.html, enter your URL and key, click Open Vault. Your vault opens. From now on it remembers your credentials — you will not be asked again.

For the Chrome extension (when available): Click the Hyphælium icon in your Chrome toolbar, enter your URL and key, click Connect.

One set of credentials — everywhere. Your URL and key are the same for the bookmarklet, the vault, and the extension. Enter them once in each place and they are remembered permanently.

Your vault is ready.

Every breadcrumb you save from this point forward lands safely in your private vault. Open it anytime at hyphaelium.com/vault.html to see your complete trail of thought.

If something went wrong

"No rows returned" after running the SQL

This is correct. It means the setup worked. Move to Step 4.

"Key rejected" when connecting

Make sure you are using the anon / public or Publishable key — not the service_role key. Copy it fresh using the copy icon rather than typing it.

"Table not found" when opening vault

The SQL in Step 3 did not run correctly. Go back to Supabase → SQL Editor → run the code block again.

"Project is paused"

Supabase free tier projects pause after 7 days with no activity. Go to supabase.com, click your project, and click Restore project. Takes about 2 minutes.
